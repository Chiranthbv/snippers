Claude Build Instructions (Copy & Use These Prompts)Use the following prompts when asking Claude to build each page. Always include the design system details above when prompting.Prompt 1 — Landing PageCopy this prompt to Claude:Build a Next.js 14 landing page for a code snippet sharing platform called "SnippetHub".Use Tailwind CSS and Framer Motion.Design system:- Background: #0A0F1E (dark navy)- Accent: #6366F1 (indigo) and #8B5CF6 (purple) gradient- Text: #F8FAFC (primary), #94A3B8 (muted)- Cards: glassmorphism (rgba white 5% bg, white 8% border, backdrop-blur)- Font: Syne (headings) + DM Sans (body) from Google FontsInclude: sticky nav, hero section with gradient heading and two CTA buttons,gradient orb background effect, feature cards section (6 features in 3-col grid),a mock snippet preview card, and a footer.Animate sections fading in on scroll with Framer Motion.Prompt 2 — Browse / Feed PageCopy this prompt to Claude:Build a Next.js page at /snippets for browsing public code snippets.Use the same dark design system (bg #0A0F1E, accent #6366F1, Tailwind CSS).Include:- Search bar at the top with icon- Filter tabs for programming language (All, JavaScript, Python, Java, etc.)- Sort dropdown (Newest, Most Viewed)- Responsive grid of snippet cards (3 col desktop, 2 tablet, 1 mobile)Each snippet card shows:- Language badge (color-coded: JS=yellow, Python=blue, Java=orange, etc.)- Title and author username with avatar- First 4 lines of code preview in monospace font- View count and created time- Copy and Open buttons on hoverFetch from: GET ${process.env.NEXT_PUBLIC_API_URL}/api/snippetsShow loading skeleton state while fetching.Prompt 3 — Snippet ViewerCopy this prompt to Claude:Build a Next.js page at /s/[shortUrl] for viewing a single code snippet.Use the dark design system (bg #0A0F1E, accent #6366F1, Tailwind CSS).Include:- Full-width code block with line numbers, syntax highlighting via Prism.js- Language badge, "Copy Code" button (swaps to checkmark on copy)- Share button that copies the short URL to clipboard- Metadata below: author, created date, view count, expires date (if set)- If snippet is expired: show an "Expired" state with an icon and message- If snippet is private and user is not owner: show 403 forbidden stateFetch from: GET ${process.env.NEXT_PUBLIC_API_URL}/api/snippets/{shortUrl}Increment view count is handled server-side automatically.Prompt 4 — Create SnippetCopy this prompt to Claude:Build a Next.js page at /snippets/new for creating a code snippet.Use the dark design system (bg #0A0F1E, accent #6366F1, Tailwind CSS).This page requires the user to be logged in (redirect to /auth/login if no JWT token).Include:- Title input field- Language dropdown (JavaScript, Python, Java, TypeScript, Go, Rust, Other)- Large code textarea in JetBrains Mono font- Visibility toggle: Public / Private (styled toggle switch)- Optional expiry date picker- "Create Snippet" submit button (gradient blue/purple)- After submit: redirect to /s/{shortUrl} of the new snippetPOST to: ${process.env.NEXT_PUBLIC_API_URL}/api/snippetsSend Authorization: Bearer {token} header.Show toast notification on success and error using React Hot Toast.Prompt 5 — Auth Pages (Login + Register)Copy this prompt to Claude:Build two Next.js auth pages: /auth/login and /auth/register.Use the dark design system (bg #0A0F1E, accent #6366F1, Tailwind CSS).Both pages:- Centered card on page (glass card, max-w-md)- Logo at top- Form fields with dark input styling, focus ring in accent color- Submit button with gradient background- Link to switch between login and registerLogin: POST ${NEXT_PUBLIC_API_URL}/api/auth/login       Store returned JWT token in localStorage as "snippet_token"Register: POST ${NEXT_PUBLIC_API_URL}/api/auth/register          On success, redirect to /auth/loginShow error messages inline (e.g. "Invalid credentials", "Email already taken").6.
