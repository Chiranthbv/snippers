Project OverviewA RESTful backend API for a code snippet sharing platform, similar to Pastebin. Users can post code snippets with syntax highlighting metadata, set them as public or private, generate shareable short URLs, and optionally set expiration times. The platform also has basic view analytics per snippet.🎯 Primary GoalLearn core Java Spring Boot backend development by building a real-world, production-like project.The completed project should be deployable, documented, and presentable to recruiters.2. Objectives & Success CriteriaLearning ObjectivesBuild and structure a Spring Boot REST API from scratchImplement JWT-based authentication and authorizationDesign a normalized relational database schema using JPA/HibernateImplement scheduled background jobs (content expiration)Generate unique short URLs using hashing / UUID strategiesWrite unit and integration testsDocument the API with Swagger / OpenAPISuccess CriteriaAll API endpoints return correct HTTP status codes and JSON responsesExpired snippets are automatically cleaned up by a scheduled taskSwagger UI is accessible and fully documents all endpointsAt least 70% unit test coverage on the service layerProject is deployed to a public URL (Railway / Render)3. Feature Requirements4.1 Authentication & AuthorizationPOST /api/auth/register — register with username, email, passwordPOST /api/auth/login — returns a signed JWT tokenJWT is required for creating, editing, or deleting snippetsPasswords hashed with BCrypt4.2 Snippet Management (Core CRUD)EndpointMethodAuthDescription/api/snippetsGETNoList all public snippets (paginated)/api/snippets/{shortUrl}GETNoGet single snippet by short URL/api/snippetsPOSTYesCreate a new snippet/api/snippets/{id}PUTYes (owner)Update your snippet/api/snippets/{id}DELETEYes (owner)Delete your snippet/api/snippets/myGETYesGet all snippets by logged-in user4.3 Snippet FieldsFieldTypeRequiredNotestitleStringYesMax 100 charscontentTextYesThe actual code/textlanguageStringNoe.g. java, python, js — for syntax highlightingvisibilityEnumYesPUBLIC or PRIVATEexpiresAtDateTimeNoNull = never expiresshortUrlStringAutoSystem-generated 6-char unique keyviewCountIntegerAutoIncrements each time snippet is fetched4.4 Short URL GenerationOn create, generate a 6-character alphanumeric code (e.g. "abc123")Use UUID + Base62 encoding or NanoID approachCollision-check against DB before savingFinal URL format: https://yourapp.com/s/abc1234.5 Expiring LinksUser sets optional expiresAt datetime on create/updateA scheduled Spring job runs every hour and deletes expired snippetsAttempting to access an expired snippet returns 410 Gone4.6 VisibilityPUBLIC — visible in the public feed and accessible without authPRIVATE — only the owner can see it (must be authenticated + owner)4.7 Search & FilterGET /api/snippets?language=java — filter by programming languageGET /api/snippets?search=algorithm — keyword search in title/contentGET /api/snippets?sort=popular — sort by view count5. Non-Functional RequirementsSecurityAll passwords stored as BCrypt hashes — never plain textJWT tokens expire after 24 hoursUsers can only edit/delete their own snippets (enforced server-side)Input validation on all endpoints — reject invalid/missing fieldsPerformancePaginate all list endpoints (default page size: 10)Add DB indexes on shortUrl, userId, expiresAt, createdAtDeveloper ExperienceSwagger UI at /swagger-ui.html — fully documents all endpointsSeed data script with 20+ sample public snippetsClean .env / application.properties config — no hardcoded secrets6. Out of Scope (v1.0)Frontend / UI — this is a backend API onlyReal-time collaborative editingComments or reactions on snippetsOAuth2 social login (GitHub, Google) — can be added in v2Rate limiting (good future addition)7.
